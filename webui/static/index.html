📚 System Risk Flags Glossary (Click to Collapse/Expand)
Heuristics & Rules (Agent 01)
VERY_HIGH_AMOUNT: Transaction size is exceptionally large (typically > 50,000 INR). Triggers immediate hold and severe compliance checks.
HIGH_AMOUNT: Transaction size exceeds standard peer-to-peer thresholds (typically > 10,000 INR), carrying elevated base risk.
HIGH_VELOCITY_COUNT: Multiple separate transactions executed in under 5 minutes, typical of automation drainage.
HIGH_VELOCITY_AMOUNT: Abnormally large total volume transferred over 5 minutes, indicating rapid linear cash siphoning.
NEW_BENEFICIARY: Money sent to a recipient added within the last 30 days. First-time links represent elevated transfer risk.
HIGH_RISK_RAIL: Payments executed using instant, irreversible pathways (like UPI or Crypto) for large amounts.
ROUND_AMOUNT: Rounded clean integer lump sums typical of automated script bots or systematic cash-out runs.
Behavioral & Identity (Agent 02)
FOREIGN_LOGIN: Login country does not match the registered home country of the customer profile.
DEVICE_MISMATCH: Device hardware fingerprint does not match any previously authorized device on profile history.
NEW_DEVICE: The physical device used is brand new and completely unrecognized in the platform database.
NEW_GEO: Login location belongs to a geographic coordinates cluster never previously recorded for this customer.
IMPOSSIBLE_TRAVEL: Logins occur at separate locations physically impossible to travel to in the elapsed time.
AMOUNT_SPIKE: Transaction value represents a huge percentage spike above average historical daily spend thresholds.
LOGIN_BURST: Multiple repeated login attempts within 10 minutes, typical of brute force credential stuffing.
ISOLATION_FOREST_OUTLIER: Advanced machine learning model flags the transaction vector as an anomalous outlier.
Graph Relational (Agent 03)
SHARED_DEVICE: Hardware is reused across multiple distinct account profiles. Strong indicator of organized mule operations.
SHARED_IP: Local network connection shared across several accounts, indicating localized cyber fraud farming centers.
MULE_HUB: Receiving account receives rapid cash flows from multiple distinct accounts, acting as a collection point.
MULE_CHAIN: Receiver immediately forwards funds downstream through consecutive hops (A → B → C) to mask traces.
FRAUD_RING: Closed cyclic loop of transactions detected (A → B → C → A) designed to keep illicit cash circulating internally.
